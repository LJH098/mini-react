# Runtime Demo Presentation Script

안녕하세요. 이번 과제에서는 React의 핵심 개념인 Component, State, Hooks를 직접 구현하고, 그 위에서 실제로 동작하는 웹 페이지를 만드는 것을 목표로 했습니다.

저희는 기존에 만들었던 Virtual DOM, Diff, Patch 구조 위에, 함수형 컴포넌트를 실행하고 상태와 hook 정보를 관리하는 내부 실행 로직을 추가하는 방식으로 구현했습니다.
핵심 아이디어는 함수형 컴포넌트를 그냥 호출하는 것이 아니라, FunctionComponent라는 클래스로 감싸서 상태와 hook 실행 정보를 직접 관리하는 것입니다.

이 클래스는 크게 세 가지 역할을 합니다.
첫 번째는 hooks 배열로 상태와 메모, effect 정보를 저장하는 것,
두 번째는 mount()로 최초 렌더링하는 것,
세 번째는 update()로 상태 변경 이후 다시 렌더링하는 것입니다.

이번 구현에서 중요한 제약도 과제 조건에 맞췄습니다.
Hook과 State는 루트 컴포넌트에서만 사용 가능하게 만들었고,
자식 컴포넌트는 상태 없이 props만 받는 stateless plain function으로 구성했습니다.
즉 상태는 모두 루트에서 관리하고, 자식은 화면 조각을 렌더링하는 역할만 하도록 만들었습니다.

이제 데모 페이지를 보여드리겠습니다.

지금 보이는 페이지가 runtime-demo.html입니다.
처음에는 로그아웃 상태이기 때문에 소개 화면과 미리보기만 보입니다.
여기서 로그인 버튼을 누르면, isLoggedIn 상태가 바뀌고 루트 컴포넌트가 다시 렌더링되면서 게시판 레이아웃으로 전환됩니다.

여기서 볼 수 있는 컴포넌트는 예를 들면 TopBar, SidebarMenu, BoardToolbar, PostList, PostDetail처럼 나뉘어 있습니다.
각 컴포넌트는 props를 받아서 화면만 그리고, 실제 데이터는 루트에서 관리합니다.

이제 글쓰기 버튼을 눌러보겠습니다.
제목과 내용을 입력하고 저장하면, 루트의 posts state가 변경됩니다.
그러면 update()가 실행되고, 새로운 Virtual DOM이 만들어집니다.
그 다음 이전 Virtual DOM과 diff를 비교해서 바뀐 부분만 patch로 실제 DOM에 반영합니다.
그래서 전체를 다시 그리는 것이 아니라 필요한 부분만 업데이트됩니다.

이 과정에서 Hooks도 같이 동작합니다.
useState는 로그인 여부, 게시글 목록, 선택된 게시글, 검색어, 정렬 상태 같은 값을 유지합니다.
중요한 점은 함수형 컴포넌트는 매번 다시 실행되지만, 상태값은 함수 바깥의 hooks 배열에 저장되기 때문에 유지된다는 점입니다.

useMemo는 파생 데이터를 계산할 때 사용했습니다.
예를 들면 카테고리별 개수, 정렬된 게시글 목록, 선택된 게시글, 요약 통계 같은 값입니다.
즉 state가 바뀔 때마다 무조건 모든 계산을 다시 하는 것이 아니라, 의존성이 바뀐 경우에만 다시 계산하도록 했습니다.

useEffect는 렌더 이후의 부수 효과를 처리합니다.
여기서는 게시글과 로그인 상태를 localStorage에 자동 저장하고, 화면 상태에 따라 document.title도 바꾸도록 구현했습니다.

좋아요도 한 번 눌러보겠습니다.
좋아요를 누르면 해당 게시글의 liked와 likes 값이 바뀌고, 다시 렌더링이 일어납니다.
그러면 목록, 상세 패널, 통계 값이 모두 같은 루트 state를 기준으로 함께 갱신됩니다.
이 부분이 바로 state를 한 곳에 모으는 Lifting State Up 방식의 장점입니다.

정리하면, 이번 과제에서는
함수형 컴포넌트를 기반으로 UI를 나누고,
State는 루트 컴포넌트에서만 관리하고,
Hooks로 상태 유지와 부수 효과, 메모이제이션을 구현했으며,
최종적으로는 Virtual DOM + Diff + Patch 흐름으로 실제 DOM을 업데이트하는 mini React를 만들었습니다.

이상입니다.
